## Blob

A **Blob** (Binary Large Object) represents immutable raw data — images, text, audio, or any binary content. It's the foundation for files, downloads, and object URLs.

## Creating a Blob

```javascript
const blob = new Blob(parts, options);
```

- `parts` — array of strings, ArrayBuffer, TypedArrays, or other Blobs
- `options` — `{ type: 'mime/type' }`

## Blob as Object URL

`URL.createObjectURL()` creates a temporary URL you can use in `<a>`, `<img>`, `<iframe>`, etc.:

```javascript
const url = URL.createObjectURL(blob);

// Use it
img.src = url;
a.href = url;

// Release when done
URL.revokeObjectURL(url);
```

## Blob as Download

Trigger a file download with a Blob:

```javascript
function download(filename, blob) {
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = filename;
  a.click();
  URL.revokeObjectURL(url);
}
```

## Blob vs ArrayBuffer

| Blob | ArrayBuffer |
|---|---|
| Immutable | Mutable (via views) |
| Has MIME type | Raw bytes |
| File/network friendly | Memory manipulation |
| `blob.text()`, `blob.arrayBuffer()` | TypedArray, DataView |

## Key Takeaways

- `new Blob([data], { type })` creates a blob with MIME type
- `URL.createObjectURL(blob)` creates a temporary URL for downloads/previews
- Always call `URL.revokeObjectURL()` to free memory
- Blobs convert easily: `await blob.text()`, `await blob.arrayBuffer()`
- `blob.slice()` extracts a portion — great for chunked uploads
- Blobs are **immutable** — you can't modify them in place
- Blobs are the foundation of **File**, **FileReader**, and **downloads**
